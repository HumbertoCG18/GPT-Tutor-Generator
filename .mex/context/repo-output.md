# Repo Output Format

This documents the structure of the tutor repository generated by the tool. These
paths are generated output paths, not files expected to exist in this source repo.
Reviewed against the current generator modules on 2026-05-12.

## Critical Generated Files

```text
course/COURSE_MAP.md                         # pedagogical map and primary tutor entry point
course/FILE_MAP.md                           # routing index with priority, unit, subtopic, and timeline hints
course/SYLLABUS.md                           # syllabus generated from SubjectProfile when available
course/GLOSSARY.md                           # glossary seeded and refined from course content
course/.timeline_index.json                  # internal timeline index used for regeneration
course/SOURCE_REGISTRY.yaml                  # source registry for processed materials
student/STUDENT_STATE.md                     # student profile, current position, and progress state
student/STUDENT_PROFILE.md                   # student profile generated when available
student/batteries/                           # per-unit learning batteries and progress folders
exercises/EXERCISE_INDEX.md                  # exercise index, when exercise entries exist
exams/EXAM_INDEX.md                          # exam index, when exam entries exist
assignments/ASSIGNMENT_INDEX.md              # assignment index, when assignment entries exist
code/CODE_INDEX.md                           # code index, when code entries exist
whiteboard/WHITEBOARD_INDEX.md               # whiteboard index, when whiteboard entries exist
content/BIBLIOGRAPHY.md                      # bibliography index, when bibliography data exists
build/claude-knowledge/bundle.seed.json      # internal seed, not loaded eagerly by tutor projects
build/PROGRESS_SCHEMA.md                     # progress schema under generated build metadata
setup/INSTRUCOES_CLAUDE_PROJETO.md           # tutor instructions for Claude
setup/INSTRUCOES_GPT_PROJETO.md              # tutor instructions for GPT
setup/INSTRUCOES_GEMINI_PROJETO.md           # tutor instructions for Gemini
system/TUTOR_POLICY.md                       # shared tutor policy
system/PEDAGOGY.md                           # shared pedagogical behavior
system/MODES.md                              # tutor mode definitions
system/OUTPUT_TEMPLATES.md                   # shared response templates
README.md                                    # generated repository readme
.gitignore                                   # generated repository ignore rules
```

## Source Modules That Generate These Files

| Source path | Responsibility |
|---|---|
| `src/builder/ops/pedagogical_regeneration.py` | Writes pedagogical, setup, system, student, and index artifacts during regeneration. |
| `src/builder/artifacts/repo.py` | Writes operational artifacts such as source registry, bundle seed, and build report. |
| `src/builder/routing/file_map.py` | Builds routing indexes, unit matching, subtopic matching, and timeline scoring used by the generated routing index. |
| `src/builder/artifacts/deeptutor.py` | Exports selected generated artifacts into the DeepTutor knowledge directory. |

## How the Tutor Uses These Files

1. On first session, the tutor starts from the generated course map and routing index under the generated course directory.
2. It maps loaded materials to course units using the generated routing index as the navigation index.
3. It checks the generated student state before explanations so answers respect the student's current progress.
4. Internal files under the generated build directory are operational metadata and should not be eagerly loaded as tutor knowledge.

## Generated File Map Format

The generated routing index is written during build. It contains:

- file paths for generated content entries
- priority weights
- unit and subtopic assignment hints
- timeline period hints where the course schedule supports them
- confidence markers for ambiguous matches

The matching and routing logic lives in `src/builder/routing/file_map.py`.

## Multi-LLM Instruction Files

The build generates separate instruction files for each supported LLM from a
single run:

- Claude Projects instructions under the generated setup directory
- GPT instructions under the generated setup directory
- Gemini instructions under the generated setup directory

These are generated from the current course metadata, student profile, subject
profile, and repository feature flags in `src/builder/ops/pedagogical_regeneration.py`.

## What Does Not Exist in the Generated Repo

- No LLM API calls happen at build time; the LLM is used only at runtime inside the generated tutor repository.
- No manual tutor-side rewrite of the generated course map or routing index; regeneration owns these files.
